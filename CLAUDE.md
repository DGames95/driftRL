# Project: Drift Toy — RL + playable 2D drifting game

A small toy game: a 2D car on a track, an RL agent, and a report. Keep it
lite and plainly written — no extra abstraction layers.

## Layout

```
track.py            Track: circle / random bounded-curvature centerlines,
                    track-frame errors (e_y, e_psi) and curvature preview
drift_env.py        DriftEnv (gymnasium.Env): physics, grip/drift reward
                    modes, analytic_grip_limit()
train.py            PPO training -> models/<mode>_<track>/best_model.zip
evaluate.py         rollout, live animation, report figures, --instability demo
game.py             pygame chase-view game (manual arrows or --demo agent)
report/report.tex   report
report/figures/     generated by evaluate.py (+ game.py --screenshot)
models/             saved SB3 models, one dir per run: {grip,drift}_{circle,
                    random}/{best_model,final_model}.zip
logs/               tensorboard runs
```

## Environment

using `gncnet` conda env (gymnasium 1.2.2, sb3 2.7.1, torch 2.5.1, numpy 2.2.6,
pygame 2.6.1). if not existing, create a new one.

## Model and environment
Values subject to more tuning

- 3-DOF bicycle model, explicit Euler dt = 0.02 s, 1000-step (20 s) episodes.
  Tires: linear with tanh saturation. The driven rear axle has a
  friction-ellipse coupling (longitudinal force use shrinks its lateral
  limit) and its longitudinal force is capped at mu*Fz_r. The coupling is
  what lets throttle kick the tail out (power-oversteer).
- Observation (8): [vx, vy, r, e_y, e_psi, kappa @ 0/10/25 m ahead], divided
  by OBS_SCALE. e_y > 0 = left of centerline; positive curvature = left turn.
  Action: [delta in +-0.5 rad, T in +-1].
- Tracks (track.py): circle R = 30 m (closed, constant curvature), or random
  open tracks, |kappa| in [1/60, 1/22], piecewise-linear knots every 30-80 m,
  length ~500 m. Half-width 4 m. Random training tracks regenerate each reset.

## Workflow

- `evaluate.py --instability` and `evaluate.py --mode {grip,drift} --no-anim`
  generate the report figures; then `pdflatex` (2 passes) in report/.
- game.py input lags: tau_steer 0.12 s, tau_throttle 0.25 s; camera follows
  velocity direction (heading below 3 m/s). `--mode {grip,drift}` selects
  which reward shaping the HUD reports.
