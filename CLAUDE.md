# Project: Drift Toy — RL + playable 2D drifting game

The requirement is a simple toy game with RL and a report documenting the
results. Nothing else is sacred: earlier prompt constraints (file count,
fixed reward form, "no conclusions in the report") have been explicitly
relaxed by the user. Keep everything lite and simply written — no extra
abstraction layers.

## Layout

```
track.py            Track: circle / random bounded-curvature centerlines,
                    track-frame errors (e_y, e_psi) and curvature preview
drift_env.py        DriftEnv (gymnasium.Env): physics, grip/drift reward
                    modes, analytic_grip_limit()
train.py            PPO training -> models/<mode>_<track>/best_model.zip
evaluate.py         rollout, live animation, report figures, --instability demo
game.py             pygame chase-view game (manual arrows or --demo agent)
report/report.tex   report story: model -> grip baseline -> drift -> game
report/figures/     generated by evaluate.py (+ game.py --screenshot)
models/             saved SB3 models; v1_no_ellipse/ = early degenerate run
logs/               tensorboard; train_*.txt = training stdout
```

## Environment

`gncnet` conda env (gymnasium 1.2.2, sb3 2.7.1, torch 2.5.1, numpy 2.2.6,
pygame 2.6.1). PPO uses `device="cpu"` — MLPs train faster on CPU in SB3.

## Model / env facts

- 3-DOF bicycle model, explicit Euler dt = 0.02 s. Tires: linear with tanh
  saturation. Rear axle has a friction-ellipse coupling (longitudinal force
  use shrinks lateral limit) and its longitudinal force is capped at
  mu*Fz_r. This coupling is what makes throttle kick the tail out; without
  it the model is a fast-cornering model, not a drifting one.
- Observation (8): [vx, vy, r, e_y, e_psi, kappa @ 0/10/25 m ahead],
  scaled by OBS_SCALE. e_y > 0 = left of centerline; positive curvature =
  left turn. Action: [delta in +-0.5 rad, T in +-1].
- Tracks (track.py): circle R = 30 m, or random open tracks with curvature
  bounded |kappa| in [1/60, 1/22] (piecewise-linear knots every 30-80 m),
  half-width 4 m. Random training tracks regenerate each reset.
- Reward modes: both have vx*cos(e_psi) - 0.5*e_y^2 - 0.002*delta_dot^2
  and a -400 termination penalty (|e_y| > 4 or vx < 1).
  drift: +3.0*|beta|.  grip: -50*beta^2.
- Analytic traction limit (circle): v^4 [(m/R)^2 + (c_d L/lf)^2] = (mu m g)^2
  -> 16.26 m/s at R = 30. The grip agent's settled speed should match this;
  that is the physics-validation argument in the report.
- PPO: lr 3e-4, gamma = 0.999, rest SB3 defaults. gamma was deliberately
  raised from 0.99: with 0.99 PPO converged to full-throttle-until-ejected
  (left track at 3.7 s). Do not lower it back.

## Lessons learned (kept for context)

- The naive reward (small terminal penalty, gamma 0.99) produced a
  degenerate speed-maximizing policy; models/v1_no_ellipse/ keeps it.
- Open-loop drift dynamics are unstable: constant inputs corner steadily,
  but a 0.5 s throttle stab spins the car (evaluate.py --instability).
- The drift agent's steady behavior is a pulsed drift limit cycle with
  steering saturating at +0.5 rad; raising W_DDOT would smooth it if ever
  needed.

## Workflow

- `evaluate.py --instability` + `evaluate.py --mode {grip,drift} --no-anim`
  generate all report figures before `pdflatex` (2 passes) in report/.
- game.py first-order input lags: tau_steer 0.12 s, tau_throttle 0.25 s;
  camera follows velocity direction (heading below 3 m/s).
